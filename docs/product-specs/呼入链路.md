# 智能客服话务平台 - 产品需求规格

> 本文档从业务视角描述话务平台的功能需求，技术实现方案见 `docs/design-docs/呼入链路工程化架构设计.md`。
> 禅道项目：「智能客服话务平台」

## 1. 业务背景

电力行业智能客服话务平台。客户拨打客服热线，由坐席接听并提供服务。系统在通话过程中提供 AI 实时辅助（语音转写、情绪识别、意图分析、坐席建议），帮助坐席更高效地处理客户诉求。通话结束后，系统结合大模型自动提取关键信息并生成工单。

部署方式：私有化部署，单体架构。

## 1.1 架构决策

- **单体应用**：基于 JeecgBoot `jeecg-system-start` 统一启动，新增 `jeecg-boot-module-call` 子模块
- **不引入微服务组件**：≤100 并发坐席，单机足够；服务边界已在进程级别划好
- **本项目只负责统一服务端**：Web 前端、freeswichService、Call_Handing_Gateway 各有团队

## 2. 用户角色

| 角色 | 说明 |
|---|---|
| 客户 | 拨打客服热线的电力用户，不直接使用本系统 |
| 坐席（接听员） | 接听来电、处理客户诉求、查看 AI 辅助信息 |
| 管理员 | 管理坐席账号、技能组配置、权限分配、系统监控 |

## 3. 核心场景

### 场景一：客户呼入 → 坐席振铃

1. 客户拨打客服热线
2. 系统根据技能组和坐席状态自动分配可接听坐席
3. 目标坐席收到来电弹屏（显示来电号码、技能组等信息）

### 场景二：坐席接听 → 通话中

1. 坐席点击接听，双方建立通话
2. 系统实时处理通话音频，产出：
   - 语音转写（双声道，区分客户/坐席）
   - 客户情绪分析
   - 客户意图识别
   - 实体提取（户号、表号、地址、手机号等电力业务实体）
   - 坐席辅助建议（话术推荐、知识库匹配、工单建议）
3. 以上结果实时推送到坐席界面

### 场景三：来电弹屏 — 客户匹配

1. 系统收到来电号码后，自动匹配客户档案
2. 弹屏展示客户信息（姓名、户号、表号、地址、历史通话记录）
3. 若未匹配到客户，显示未知来电，坐席可在通话中手动关联或新建客户

### 场景四：通话结束 → 工单生成

1. 任一方挂断，通话结束
2. 系统生成通话摘要（要点、待办事项、客户意图总结）
3. 大模型自动提取关键信息，预填工单字段（客户信息、问题摘要、优先级）
4. 坐席确认/修改后提交工单
5. 工单与通话记录关联

### 场景五：管理员配置

1. 管理员创建/编辑坐席账号
2. 管理员配置技能组及坐席分配规则
3. 管理员管理角色权限
4. 管理员管理客户档案

## 4. 功能清单

### 4.1 坐席侧功能

| 功能 | 优先级 | 说明 |
|---|---|---|
| 登录/登出 | P0 | JWT 认证，支持 Token 刷新 |
| 坐席状态切换 | P0 | 在线/休息/忙碌，影响来电分配 |
| 来电弹屏 | P0 | WebSocket 推送振铃通知 |
| 接听/挂断 | P0 | 坐席操作入口 |
| 实时转写展示 | P0 | 区分客户/坐席双声道 |
| 情绪/意图/实体展示 | P0 | AI 分析结果实时推送 |
| 坐席辅助建议 | P0 | 话术推荐、知识库匹配 |
| 通话摘要 | P0 | 挂断后自动生成 |
| 工单创建 | P0 | 通话结束后大模型辅助生成工单 |
| 客户信息查看 | P0 | 来电弹屏展示匹配到的客户档案 |
| 通话记录列表 | P0 | 分页查询历史通话，筛选条件（时间/坐席/状态） |
| 通话记录详情 | P0 | 完整转写文本、情绪、意图、实体、摘要、标签 |
| 录音回放 | P0 | 播放录音 + 文本同步定位 |
| 通话标签 | P0 | 自动生成 + 手动编辑 |

### 4.2 管理员功能

| 功能 | 优先级 | 说明 |
|---|---|---|
| 坐席账号管理 | P0 | CRUD，基于 JeecgBoot 用户体系 |
| 技能组管理 | P0 | 配置技能组、分配坐席 |
| 角色权限管理 | P0 | 基于 JeecgBoot RBAC |
| 客户档案管理 | P0 | 客户信息 CRUD，支持多联系方式 |
| 工单管理 | P0 | 工单列表、状态流转、分配 |
| 系统监控 | P1 | 在线坐席数、通话中数量等 |

### 4.3 服务端核心能力（本项目开发范围）

| 能力 | 说明 |
|---|---|
| 账号与权限 | JWT 签发/刷新/黑名单、RBAC 鉴权 |
| 呼入路由 | 根据技能组 + 坐席状态分配坐席，分布式锁防重复分配 |
| 通话会话管理 | 创建/更新/结束通话会话，维护 call_session + call_participant + call_turn |
| 音频接收与存储 | 接收 freeswichService 上行的 WAV 语音段（已完成解码/分离/VAD/封装），存储到 MinIO |
| AI 编排 | 调用 Call_Handing_Gateway 的 ASR/NLP/AgentAssist/摘要接口，维护通话全量上下文 |
| 实时推送 | WebSocket 连接管理、事件下发（振铃/转写/分析/结束） |
| 客户档案 | 客户信息 CRUD、来电号码匹配、多联系方式管理 |
| 工单管理 | 工单 CRUD、大模型辅助填充、状态流转、与通话关联 |
| 存储访问 | DB/Redis/MinIO 统一管理，其他服务不直连 |
| 内部接口 | 供 freeswichService 调用的路由申请、事件上报、音频上行接口 |

## 5. 非功能需求

| 维度 | 要求 |
|---|---|
| 并发规模 | ≤100 坐席同时在线 |
| 响应时间 | API 接口 <4s（人类等待极限），实时推送尽量 <2s |
| 部署 | 私有化，单机或小集群 |
| 可用性 | 通话中服务端崩溃可恢复（基于 DB/Redis 状态重建） |
| 安全 | JWT + RBAC，内部服务间 X-Internal-Key 认证 |
| 存储 | 通话音频保留（MinIO），通话记录持久化（DB） |

## 7. JeecgBoot 影响面清单

### 后端（本项目开发范围）

- Controller: 呼入路由、通话会话、坐席状态、音频接收、WebSocket、客户档案、工单
- Service: 路由分配、会话管理、音频处理、AI 编排、推送、客户匹配、工单生成
- Mapper/XML: call_session、call_participant、call_turn、agent_profile、skill_group、customer、work_order
- DTO/Entity: CallSession、CallParticipant、CallTurn、AgentProfile、SkillGroup、Customer、WorkOrder
- 数据库: 通话域 + 坐席域 + 客户域 + 工单域 + 预留表

### 前端（其他团队负责，此处仅列出接口契约）

- 页面: 坐席工作台（来电弹屏、转写面板、辅助建议）
- API: WebSocket /ws/call-events、REST /api/v1/calls/*
- 路由: 新增坐席工作台页面由

### 系统配置（JeecgBoot 特有）

- [ ] 菜单权限配置（sys_permission 表）
- [ ] 按钮权限编码（坐席状态切换/接听/挂断）
- [ ] 数据权限（坐席只能看自己的通话记录）

### 旁路

- 消息通知: WebSocket 实时推送（非传统消息通知）
- 日志记录: 通话事件全链路日志
- 缓存失效: 通话结束后清理 Redis 热缓存

## 8. 与其他团队的接口边界

本项目（统一服务端）对外暴露的接口：

| 对接方 | 接口类型 | 说明 |
|---|---|---|
| Web 前端 | REST + WebSocket | 登录、坐席操作、实时事件推送 |
| freeswichService | REST + WebSocket | 路由申请、事件上报、音频帧上行 |
| Call_Handing_Gateway | REST | AI 能力调用（ASR/NLP/AgentAssist/摘要） |

本项目不直接对接 FreeSWITCH、AI 模型集群。
