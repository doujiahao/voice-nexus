## 架构决策

- **单体应用**：基于 `jeecg-system-start` 统一启动，新增 `jeecg-boot-module-call` 子模块
- **不引入微服务组件**：不使用 Spring Cloud / Nacos / Gateway
- **数据库设计兼顾扩展**：呼出、IVR、质检等预留表结构，通话方向用 direction 字段区分

## 数据库设计要点

### call_session 核心字段

| 字段 | 说明 | 扩展考虑 |
|---|---|---|
| direction | INBOUND / OUTBOUND | 呼出复用同一张表 |
| status | QUEUING / RINGING / ANSWERED / HOLDING / TRANSFERRING / ENDED | 预留转接/保持 |
| customer_id | 关联客户档案 | 来电匹配 |
| work_order_id | 关联工单 | 通话结束后填充 |
| ivr_flow_id | 预留 IVR 入口 | 本期为 NULL |
| campaign_id | 预留外呼活动关联 | 本期为 NULL |
| hangup_cause | 挂断原因枚举 | |
| queue_enter_time | 进入排队时间 | 排队统计 |

### call_participant 核心字段

| 字段 | 说明 | 扩展考虑 |
|---|---|---|
| participant_type | CUSTOMER / AGENT | 预留 SUPERVISOR / TRANSFER_TARGET |
| role | INITIATOR / RECEIVER | 预留 MONITOR / COACH |
| join_time / leave_time | 参与时段 | 支持多方通话时间线 |

### call_tag 核心字段

| 字段 | 说明 |
|---|---|
| session_id | 关联通话会话 |
| tag_name | 标签名称 |
| source | AUTO / MANUAL（自动生成或手动编辑） |
| confidence | 自动标签的置信度（手动为 NULL） |

### 通话状态枚举（含预留）

```
QUEUING → RINGING → ANSWERED → TALKING → ENDED
                                  ↓ (预留)
                              HOLDING → TALKING
                              TRANSFERRING → ENDED (新 session)
                              CONFERENCING → TALKING
```

### 来电排队设计

- 所有坐席忙时，路由接口返回 `route_action=QUEUE`
- 服务端维护 Redis 排队队列（按技能组分队列，FIFO）
- 坐席空闲时自动从队列取出下一个来电，触发振铃
- 排队超时后通知 freeswichService（由 FS 侧决定播放提示音或挂断，本期预留）

### AI 编排上下文策略

- Gateway 已改为完全无状态，所有上下文由服务端维护
- 每次调用 NLP / AgentAssist 时，传入**当前 session 的全部 turns**
- 电网场景通话短、对话简单，全量传输无性能问题
- 活跃通话的 turns 缓存在 Redis，通话结束后落库并清理

## 技术决策

| 决策点 | 选择 | 理由 |
|---|---|---|
| 部署模式 | 单体（jeecg-system-start） | ≤100 坐席，私有化，运维简单 |
| WebSocket 框架 | Spring WebSocket | JeecgBoot 已集成 Spring |
| 音频接收格式 | Protobuf AudioFrame（data=WAV） | freeswichService 已完成解码/分离/VAD/封装 |
| 分布式锁 | Redisson | API 友好，JeecgBoot 生态常用 |
| Protobuf | protobuf-java | 音频帧协议，与 freeswichService 对齐 |
| MinIO SDK | minio-java | 官方 SDK |
| 工单生成 | 大模型 Prompt + Gateway 摘要接口 | 自动提取关键信息填充工单 |
| AI 上下文 | 当前 session 全量 turns | 电网场景通话短，无性能问题 |
